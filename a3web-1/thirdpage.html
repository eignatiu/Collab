Our Solution

A Breakdown of our solution and a Demo

Main Solution

We have created a series of surveys, public, field crew and supervisor validation surveys all in unison to create a request and maintenance workflow. This begins with a request being received from the public. This creates a data point and basic request information. This request then gets processed by the supervisor and updated onto a field crew map layer showing the tree point indicating there is to be work done to the tree. From there the field crew/arborists are to look at the tree to determine its conditions and next steps for maintenance. Uniting another survey which finally gets update once more creating a centralized tree inventory with the public requests being processed by supervisor and field crew.

Limitations to our Solution

Some Limitations Include

  • Filed Maps need to be adaptable for offline use using the basemap
  • Basemap is inaccurate to a degree because of the quality of parcel and building data
  • Some objects could be misinterpreted as trees
  • Detecting objects from imagery is a time consuming and system-intensive process that may require multiple iterations

Remote Sensing

    Trees have been extracted from high-resolution (0.15m X 0.15m) aerial imagery by leveraging a deep learning tool with the ArcGIS pre-trained model for tree detection. This has been achieved by optimizing the object detection performance through numerous iterations of model parameter adjustments. Buildlings and cars were also extracted similarly to aid the process of tree detection.
